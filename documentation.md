## 1. System Schemas & Data Structures

### 1.1 Cloud Blob Manifest (Azure Storage)
The application relies dynamically on objects stored in Azure Container (`AZURE_CONTAINER_NAME=zerodaydata` or `news`).

#### `issue_YYYY-MM-DD.json` & `latest.json`
The primary daily artifact generated by `scraper/pipeline.py`.
```json
{
  "date": "2026-04-18",
  "top_stories": [
    {
      "title": "Hackers exploit new vulnerability in standard router OS",
      "category": "Zero-Day", /* Forced DeepSeek mapping */
      "short_summary": "Brief 1-sentence descriptor generated by LLM.",
      "deep_summary": "Multi-paragraph insight containing Core Threat, Insight, Takeaway.",
      "score": 18, /* Integer weighting from rank_article() */
      "source": "RSS Scraping",
      "url": "https://example.com/sec-news"
    }
  ],
  "cves": [
    {
      "title": "Vulnerability CVE-2026-1234",
      "summary": "AI generated breakdown of the CVE description.",
      "cve_ids": ["CVE-2026-1234", "CVE-2026-1235"], /* Extracted via regex */
      "score": 10 /* Optional integer */
    }
  ]
}
```

#### `subscribers.json` & `subscribers_backup.json`
Used via `lib/blob_store.py` across web application ingestion and bot issuance runs.
```json
[
  {
    "email": "user@domain.com",
    "verified_email": true,
    "is_active": true,
    "verification_token": "abcd1234efgh5678",
    "verification_token_created_at": "2026-04-18T12:00:00Z",
    "unsubscribe_token": "jklm9012nopq3456",
    "created_at": "2026-04-17T08:30:00Z"
  }
]
```
*(Note: `subscribers_backup.json` acts as an append-only archive and simply contains a list array of email strings `["user@domain.com", ...]`).*

## 3. Explicit Algorithmic Flow & Filtering

### 3.1 LLM Categorization Restrictions (`scraper/categorizer.py`)
Articles dumped by `newspaper3k` enter a geometric retry-decorator mechanism `@rate_limit_and_retry(max_retries=3, base_delay=2.0)`.
- **System Prompt Constraint**: The DeepSeek model is algorithmically bound to output *only* one token matching: `CVE`, `Malware`, `Ransomware`, `Data Breach`, `Zero-Day`, `Security Tools`, `General Security`, `Artificial Intelligence`, `Computer Science`, `Tech News`.
- **Validation**: If standard substring checks (`valid_category.lower() in raw_category.lower()`) yield zero strict matches within the 10 outputs, the framework forcefully cascades back to `"General Security"`.

### 3.2 Deduplication Constraints (`scraper/pipeline.py`)
To prevent duplicate content bridging days:
1. **Internal Cache Build**: The bot fetches yesterday's issue explicitly (checking disk, then `AZURE_CONTAINER_NAME`) and populates `seen_urls` and `seen_titles`.
2. **Phase 1: Hard Length**: Rejects raw strings with length `< 200` characters.
3. **Phase 2: Hard URL**: Drops elements perfectly matching the `seen_urls` absolute string.
4. **Phase 3: Deep Fuzzy Validation**: Casts items through `difflib.SequenceMatcher(None, title1, title2).ratio() > 0.8`. Any article hitting `80%` title similarities sequentially is flagged `is_dup = True` and deleted from runtime array.

### 3.3 Dynamic Regex Intercepts (CVE Extraction)
If a string triggers categorization as purely `CVE`, the article is branched forcefully from standard news arrays:
- Skips Deep Summary routines instantly.
- Passes through python regex rules engine: `re.findall(r'CVE-\d{4}-\d{4,7}', content, re.IGNORECASE)` to extract all definitive CVE IDs mathematically into a unique list array, pushed into the explicit `cves` bucket.

### 3.4 Azure Rebound Tolerance
Any attempt pushing final execution states to Azure utilizes an isolated wrapper: `_retry_storage()`.
- **Mathematical Delay**: Executes a strict geometric fallback `delay = 2 ** attempt`. (Attempt 1: 2s; Attempt 2: 4s; Attempt 3: 8s).
- Total failure results in a local `data/output/YYYY-MM-DD/newsletter.txt` preservation artifact, logging `[ERROR][STORAGE] cloud upload failed permanently`.

## 4. Automation Orchestrations (`automation/send_newsletter.py`)

### 4.1 Security Abstractions
Outbound marketing mail is never generated raw. Before Resend API triggering:
1. Validates standard domain via `resend.Domains.list()` checking for `verified` constraints dynamically to avoid DKIM/SPF rejection drops.
2. Extracts payload logic directly utilizing an internal instance of `lib.content.get_latest_issue()`, completely bypassing Azure requests using internal memory.
3. **Generates Tracking Hash**: A unique tracking URL-safe string `secrets.token_urlsafe(16)` is allocated independently to every subscriber targeting email distributions, injected as `track_token` natively to SQLite.
4. **Humanize Invocation**: `lib.humanizer.humanize_email()` manipulates phrasing securely utilizing a `safety_filter()` module to prevent LLM hallucination overrides (blocking generic list layouts or repetitive URL dumping).

## 5. Deployment Hardware Limitations (`runbot.sh`)
When instantiated via automated CLI pipelines, Docker inherently protects the web-server execution limits explicitly scaling physical RAM allocations.
```bash
docker run -d \
   --name zeroday_bot \
   --cpus="1.0" \
   --memory="2048m" \
   -v $(pwd)/data:/app/data \
   zeroday_bot
```
The localized `data/` branch is volumed `-v` out of the execution layer securely binding `subscribers.db` and output traces to persistent host memory preventing loss during image rebuilding paradigms.
